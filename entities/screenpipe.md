# screenpipe

screenpipe is a local-first workflow-memory tool. Its homepage describes continuous screen and audio capture, search across work context, and event-triggered agents that can write notes or update other systems. In this wiki, it belongs under [[external-memory]] and [[work-routines]], not as an ADHD-specific treatment.

## ADHD-relevant pattern

The useful pattern is work memory: instead of relying on a person to remember what happened in meetings, apps, handoffs, and task switches, screenpipe tries to make that context searchable. For ADHD, this can support [[working-memory]], [[executive-function]], and [[task-initiation]] when the failure mode is "I know I did or heard something, but I cannot reconstruct it now."

## Automation pattern

The homepage's meeting-notes example triggers after a meeting, writes a short summary, action items per person, decisions, and next steps, then saves the result locally. That is a concrete [[work-routines]] pattern: convert an event into a follow-up artifact before the user has to remember to do it.

## Privacy risk

Continuous screen and audio capture is unusually sensitive. The homepage says screenpipe can exclude apps, windows, and URLs before they reach disk, and can scrub card numbers, SSNs, and keys on-device. For ADHD use, this privacy layer should be treated as part of the tool, not a nice-to-have setting.

Do not store private captures, transcripts, credentials, health information, or third-party personal data in this wiki. Keep only product-level descriptions and reusable patterns.
