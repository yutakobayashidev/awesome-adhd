# 好奇心・報酬・記憶

好奇心・報酬・記憶の研究は、ADHDそのものの治療研究ではないが、「興味がある時だけ異常に覚えられる／動ける」という体験を設計面から考える補助線になる。

Kangらの研究は、知りたい気持ちが高い問題を見た時に報酬予測に関わる尾状核領域の活動が高まり、好奇心が高い項目では記憶がよくなることを示した研究として取り込む。^[raw/papers/kang-2009-curiosity-reward-memory.md]

Kobayashi & Hsuは、報酬価値と情報価値が脳内で共通した主観価値信号として扱われうることを示す。ADHD支援では、情報・進捗・次の手がかりを「報酬っぽく」設計する発想につながるが、ADHD症状を直接説明する証拠ではない。^[raw/papers/kobayashi-2019-reward-information-value.md]

## 実装上の含意

- [[task-initiation]]では、退屈な作業を「次に何が分かるか」「小さな進捗が見えるか」へ変換すると始めやすくなる可能性がある。
- [[work-routines]]では、完了報酬だけでなく、途中の情報・発見・可視化を足場にする。
- [[hyperfocus-control]]では、興味と報酬が強すぎる作業ほど終了条件を先に置く必要がある。
